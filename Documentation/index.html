Setup

To begin, drag the index.html file into a browser such as chrome, the website should appear. Then drag the project files into a text editor such as Sublime text. You will see all your files in the side bar.


Step 1 - Adding your Behance Feed

Go into the index.html file in the Amari folder and locate this line of code in portfolio section within the script tag (you can also press command + f to search) 

window.behance = {clientId: 'dylvP53sMdPu1Nam5qvGKgvyWhfetwuS', username: 'chrisvenning'}

Then replace the behance username with your own like below

window.behance = {clientId: 'dylvP53sMdPu1Nam5qvGKgvyWhfetwuS', username: 'arielbeninca'}

Your own behance feed will now show up.

Optional: If you would like to you can generate your own Client ID/ API through this link https://www.behance.net/dev/register

Step 5 - Launching your website

Once you have completed your website it's time to launch it live. The first thing you will need is hosting, like Go Daddy. When buying 12 months hosting from go daddy it comes with a free domain name. If you already have a domain name you will have to connect it to the hosting.

Now that your hosting and domain is all setup, download a ftp program like Filezilla and connect it to your hosting. For more details on how to do that please Click Here. Once connected to your hosting through ftp, drag and drop your website files into the correct folder on Filezilla and it will upload them to your domain.

You have now completed basic customisation of the website, but remeber there are no limitations with what you can do

CSS Structure

  • normalize.css - A small CSS file that provides better cross-browser consistency in the default styling of HTML elements. It's a modern, HTML5-ready, alternative to the traditional CSS reset. Think of it as resetting all defult browser styles.
  • font-awesome.min.css - File containing the icon font css. Font Awesome gives you scalable vector icons that can instantly be customized — size, color, drop shadow, and anything that can be done with the power of CSS.
  • animate.css - a bunch of cool, fun, and cross-browser animations. Great for emphasis, home pages, sliders, and general just-add-water-awesomeness.
  • main.css - This file contains all the styles and have been commented out by section.

Sass

Our theme has been built using Sass. This means you can quite easily add your own markup and change the theme to fit your project. We recommend that you use Sass files instead of pure CSS files if you intend to apply style changes. Using sass to output a compressed CSS file will also optimise site load time.

To learn more about sass, what it is, why it's awesome, and how to use it please click here watch this video

All the Sass files can be found in the scss folder.

SVG images

How to add an "Inline SVG" image

If you check the body of every HTML document, you will see that at the top of it we added the SVG code. It may look a bit of a mess at first, but the code is just compressed to save bandwidth.

All you will need to do is include your new SVG inside the svg element. We suggest that you add it via the symbol element, so you will also be able to add a title to it, which will make it more SEO friendly.

Don't forget to add an ID to your new SVG image, which you will use to add the SVG image on your page. We prefixed all the IDs of our SVG images with shape-.
